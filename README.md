## 一、前言

排序（Sorting）是計算機科學中最基礎且重要的問題之一，  
其目的在於將一組資料依照特定順序（例如由小到大）重新排列。

不同排序演算法在設計理念、執行效率與記憶體使用上皆有所差異，  
當資料規模增加時，演算法效能的差距也會被明顯放大。

本次作業透過實作並視覺化五種常見排序演算法：

- 氣泡排序（Bubble Sort）
- 選擇排序（Selection Sort）
- 插入排序（Insertion Sort）
- 合併排序（Merge Sort）
- 快速排序（Quick Sort）

除了說明其基本原理與複雜度分析外，  
也透過實際程式模擬不同資料規模下的排序時間，  
比較各演算法在實務應用中的效能差異。

## 二、排序演算法原理說明

### （一）氣泡排序法（Bubble Sort）

氣泡排序是一種概念最直觀的排序方法。  
其核心作法是反覆比較相鄰的兩個元素，  
若順序錯誤便進行交換，  
使較大的數值逐漸往陣列右端移動，如同氣泡向上浮動。

排序過程中，每一回合都會將目前尚未排序區域中的最大值放到正確位置。

**特點：**
- 實作簡單、容易理解
- 不適合大資料量排序

---

### （二）選擇排序法（Selection Sort）

選擇排序的作法是在每一輪排序中，  
從尚未排序的資料區段中找出最小值，  
再與該區段的第一個元素進行交換。

不論資料原本是否幾乎已排序完成，  
比較次數皆固定，因此時間複雜度不會改善。

**特點：**
- 交換次數少（最多 n − 1 次）
- 比較次數固定

---

### （三）插入排序法（Insertion Sort）

插入排序的概念類似於整理撲克牌。  
它會將陣列分成「已排序區域」與「未排序區域」，  
每次從未排序區取出一個元素，  
插入到左側已排序區域的正確位置。

當資料原本接近排序完成時，  
所需移動與比較次數會大幅減少。

**特點：**
- 適合小型資料或近乎排序完成的資料
- 實務中常作為其他排序法的輔助

---

### （四）合併排序法（Merge Sort）

合併排序是一種典型的「分治法（Divide and Conquer）」演算法。  
其做法是先將資料不斷拆分成更小的子陣列，  
直到子陣列只剩一個元素後，再逐步合併成有序陣列。

在合併過程中，會同時比較左右子陣列的元素大小，  
並依序放入暫存陣列中。

**特點：**
- 效能穩定
- 適合處理大量資料
- 需要額外記憶體空間

---

### （五）快速排序法（Quick Sort）

快速排序同樣使用分治法，  
會先選擇一個基準值（Pivot），  
將陣列重新排列，使小於 Pivot 的元素在左側，  
大於 Pivot 的元素在右側。

接著對左右子陣列重複進行相同操作，直到排序完成。

**特點：**
- 平均效能極佳
- Pivot 選擇不當時效能會惡化

## 三、複雜度分析

### （一）時間複雜度分析

| 排序方法 | 最佳情況 | 平均情況 | 最差情況 |
|--------|--------|--------|--------|
| 氣泡排序 | O(n) | O(n²) | O(n²) |
| 選擇排序 | O(n²) | O(n²) | O(n²) |
| 插入排序 | O(n) | O(n²) | O(n²) |
| 合併排序 | O(n log n) | O(n log n) | O(n log n) |
| 快速排序 | O(n log n) | O(n log n) | O(n²) |

---

### （二）空間複雜度分析

| 排序方法 | 空間複雜度 | 是否需額外記憶體 |
|--------|----------|--------------|
| 氣泡排序 | O(1) | 否 |
| 選擇排序 | O(1) | 否 |
| 插入排序 | O(1) | 否 |
| 合併排序 | O(n) | 是 |
| 快速排序 | O(log n) | 是（遞迴堆疊） |

## 四、實驗設計與數據模擬

### （一）實驗目的

透過實際程式執行五種排序演算法，  
比較在不同資料規模（n 值）下的執行時間，  
驗證理論時間複雜度與實際效能表現之差異。

---

### （二）實驗設定

- 使用隨機產生之整數陣列
- 每一種排序法皆使用相同資料內容進行測試
- 測試資料量：
  - n = 100
  - n = 1,000
  - n = 5,000
  - n = 10,000
- 取多次實驗平均結果以降低誤差
