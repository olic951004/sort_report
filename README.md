## 氣泡排序法（Bubble Sort）

**氣泡排序法**是一種概念簡單、容易理解的排序方法。  
它會重複比較相鄰的兩個元素，如果順序錯誤就進行交換，  
讓較大的數值像「氣泡」一樣慢慢往陣列右側浮動。

在本次實作中，我透過長條圖的方式呈現陣列內容，  
並在排序過程中即時顯示元素交換的情況，使排序過程一目了然。

- 時間複雜度：
  - 最佳情況：O(n)（資料已接近排序完成）
  - 平均 / 最差情況：O(n²)
- 優點：實作簡單、容易理解
- 缺點：當資料筆數變多時效率較差

## 選擇排序法（Selection Sort）

**選擇排序法**的核心概念是：  
在每一輪排序中，從尚未排序的區域找出最小值，  
再與該區域的第一個元素進行交換。

在我的視覺化頁面中，  
會清楚標示「目前最小值」與「正在比較的元素」，  
讓每一輪為什麼會交換變得非常直觀。

- 時間複雜度：
  - 最佳 / 平均 / 最差情況皆為 O(n²)
- 優點：交換次數少（最多 n−1 次）
- 缺點：比較次數固定，效能不佳

## 插入排序法（Insertion Sort）

**插入排序法**的作法類似整理撲克牌：  
將元素一個一個插入到左側已排序的區域中，  
直到整個陣列完成排序。

透過動畫顯示元素「往右移動」與「插入正確位置」，  
可以清楚看出插入排序在幾乎已排序資料時的高效率。

- 時間複雜度：
  - 最佳情況：O(n)
  - 平均 / 最差情況：O(n²)
- 優點：對於小資料量或近乎排序完成的資料效果佳
- 缺點：資料量大時效率仍有限

## 合併排序法（Merge Sort）

**合併排序法**是一種典型的「分治演算法（Divide and Conquer）」。  
其做法是先將資料不斷拆分成更小的子陣列，  
各自排序完成後再逐步合併成完整的有序陣列。

在本專題中，合併排序是第一個使用「遞迴概念」的排序法，  
也讓我更清楚理解分治法的運作方式。

- 時間複雜度：
  - 最佳 / 平均 / 最差：O(n log n)
- 優點：效能穩定、排序速度快
- 缺點：需要額外的記憶體空間

---

## 快速排序法（Quick Sort）

**快速排序法**同樣使用分治法，  
會先選擇一個「基準值（Pivot）」，  
再將資料分成小於與大於 Pivot 的兩部分，  
並針對子陣列重複執行相同的操作。

在視覺化頁面中，我特別標示 Pivot 與交換過程，  
方便觀察排序如何逐步完成。

- 時間複雜度：
  - 平均：O(n log n)
  - 最差：O(n²)
- 優點：實務上常見、平均效能快速
- 缺點：基準值選擇不佳時效能下降

## 綜合比較

| 排序方法 | 平均時間複雜度 | 是否使用分治法 | 特點 |
|--------|--------------|--------------|------|
| 氣泡排序 | O(n²) | 否 | 概念最簡單 |
| 選擇排序 | O(n²) | 否 | 交換次數少 |
| 插入排序 | O(n²) | 否 | 適合近乎排序完成的資料 |
| 合併排序 | O(n log n) | 是 | 效能穩定 |
| 快速排序 | O(n log n) | 是 | 平均速度最快 |
