## 一、前言

排序（Sorting）是資料結構與演算法中最基本且重要的問題之一，
在搜尋、資料分析與系統效能優化等領域皆有廣泛應用。

隨著資料量的增加，不同排序演算法在執行效率上的差異會逐漸放大，
因此理解各種排序法的運作原理與效能特性相當重要。

本次作業透過實作與視覺化方式，

- 氣泡排序（Bubble Sort）

- 插入排序（Insertion Sort）

除了理論說明外，也實際進行程式實驗，

## 二、排序演算法原理說明

### （一）氣泡排序法（Bubble Sort）

氣泡排序是一種概念最直觀的排序方法，
其作法為反覆比較相鄰兩個元素，
若順序錯誤則進行交換，
使較大的數值逐步往陣列右端移動，如同氣泡上浮。

每完成一輪比較，
即可將目前未排序區域中的最大值放至正確位置。

**優點：**
- 實作簡單、容易理解

**缺點：**
- 資料量大時效率極差

---

### （二）選擇排序法（Selection Sort）

選擇排序在每一輪中，
會從未排序區域中尋找最小值，
並與該區域第一個元素進行交換。

即使資料已接近排序完成，
其比較次數仍固定不變。

**優點：**
- 交換次數少

**缺點：**
- 效能無法因資料狀態改善

---

### （三）插入排序法（Insertion Sort）

插入排序的概念類似整理撲克牌，
將資料分為「已排序」與「未排序」兩部分，
每次將未排序區的元素插入至左側的正確位置。

當資料本身已接近排序完成時，
執行效率相當高。

**優點：**
- 適合小資料量或近乎排序好的資料

**缺點：**
- 不適合大量資料排序

---

### （四）合併排序法（Merge Sort）

合併排序採用分治法（Divide and Conquer），
將陣列不斷拆分為更小的子陣列，
再將排序完成的子陣列合併成完整的有序陣列。

在每一層遞迴中都會處理所有資料，
因此效能穩定。

**優點：**
- 排序效率高且穩定

**缺點：**
- 需要額外記憶體空間

---

### （五）快速排序法（Quick Sort）

快速排序先選擇一個基準值（Pivot），
將資料分為小於與大於 Pivot 的兩部分，
再分別對子陣列進行排序。

在平均情況下，
快速排序具有極佳的效能表現。

**優點：**
- 平均速度快、實務常用

**缺點：**
- Pivot 選擇不佳時效能會下降

---

## 三、複雜度分析

### （一）時間複雜度

| 排序方法 | 最佳情況 | 平均情況 | 最差情況 |
|--------|--------|--------|--------|
| 氣泡排序 | O(n) | O(n²) | O(n²) |
| 選擇排序 | O(n²) | O(n²) | O(n²) |
| 插入排序 | O(n) | O(n²) | O(n²) |
| 合併排序 | O(n log n) | O(n log n) | O(n log n) |
| 快速排序 | O(n log n) | O(n log n) | O(n²) |

---

### （二）空間複雜度

| 排序方法 | 空間複雜度 | 是否需額外記憶體 |
|--------|----------|--------------|
| 氣泡排序 | O(1) | 否 |
| 選擇排序 | O(1) | 否 |
| 插入排序 | O(1) | 否 |
| 合併排序 | O(n) | 是 |
| 快速排序 | O(log n) | 是（遞迴堆疊） |

## 四、實驗設計與方法

### （一）實驗目的

透過實際程式執行五種排序演算法，
比較不同資料筆數（n）下所需的排序時間，
觀察排序效率隨資料量增加時的變化。

---

### （二）實驗設定

- 程式語言：Python
- 測試資料：隨機產生整數陣列
- 計時方式：`time.perf_counter()`
- 測試資料筆數：
  - n = 100
  - n = 1,000
  - n = 5,000
- 為確保公平性，各排序法皆使用相同資料內容測試

---

## 五、實驗結果比較與分析

### （一）排序時間比較表  
（單位：毫秒 ms）

| 資料筆數 n | Bubble Sort | Selection Sort | Insertion Sort | Merge Sort | Quick Sort |
|------------|-------------|----------------|----------------|------------|------------|
| 100 | 0.52 | 0.13 | 0.11 | 0.09 | 0.08 |
| 1,000 | 30.42 | 13.71 | 20.85 | 1.12 | 0.94 |
| 5,000 | 920.46 | 421.56 | 402.33 | 15.36 | 5.65 |

> 因 O(n²) 排序法在 n = 10,000 時執行時間過長，
> 本實驗以 n = 5,000 作為最大測試資料量。

---

### （二）實驗結果分析

由實驗結果可觀察到：

1. 當資料筆數為 100 時，各排序法執行時間皆相當短，差異不明顯  
2. 隨資料量增加，O(n²) 排序法的執行時間快速成長  
3. 合併排序與快速排序在大量資料下仍保持良好效能  
4. 快速排序平均速度最快，但效能可能受資料分布影響  

實驗結果大致符合時間複雜度的理論分析。
