## 一、前言

排序（Sorting）是資料結構與演算法中最基本且重要的主題之一，
在搜尋、資料分析與系統效能優化等應用情境中皆扮演關鍵角色。

由於不同排序演算法在設計理念、效率與記憶體使用上皆有所差異，
當資料數量增加時，各演算法的效能差距也會被明顯放大。

本次作業透過實作並視覺化五種常見排序演算法：

- 氣泡排序（Bubble Sort）
- 選擇排序（Selection Sort）
- 插入排序（Insertion Sort）
- 合併排序（Merge Sort）
- 快速排序（Quick Sort）

除了說明其基本原理與複雜度分析外，
也藉由實際程式模擬不同資料規模下的排序時間，
比較各排序法在實務應用中的效能差異。

## 二、排序演算法原理說明

### （一）氣泡排序法（Bubble Sort）

氣泡排序是一種概念最直觀的排序方法，
其作法為反覆比較相鄰的兩個元素，
若順序錯誤則進行交換，
使較大的數值逐步往陣列右端移動，如同氣泡上浮。

每完成一輪比較後，
當前尚未排序區域中的最大值會被放到正確位置。

**優點：**
- 實作簡單、容易理解

**缺點：**
- 當資料量增加時效能明顯下降

---

### （二）選擇排序法（Selection Sort）

選擇排序的核心概念是在每一輪排序中，
從尚未排序的資料區域中找出最小值，
再與該區域的第一個元素進行交換。

此方法的比較次數固定，
不會因資料已部分排序而改善效能。

**優點：**
- 交換次數少（最多 n − 1 次）

**缺點：**
- 效能不隨資料狀態改變

---

### （三）插入排序法（Insertion Sort）

插入排序的運作方式類似整理撲克牌，
將資料分為「已排序」與「未排序」兩部分，
每次從未排序區取一個元素，
插入至左側已排序區的正確位置。

當資料本身接近排序完成時，
此方法的效率會大幅提升。

**優點：**
- 適合小資料量或近乎排序好的資料

**缺點：**
- 面對大量資料時效能有限

---

### （四）合併排序法（Merge Sort）

合併排序是一種典型的分治演算法（Divide and Conquer），
會先將陣列不斷切割成更小的子陣列，
直到每個子陣列只剩一個元素為止，
再將其依序合併成完整的有序陣列。

合併排序在每一層遞迴中皆處理所有資料，
因此效能相當穩定。

**優點：**
- 排序時間穩定、效率高

**缺點：**
- 需要額外的記憶體空間

---

### （五）快速排序法（Quick Sort）

快速排序同樣利用分治法，
會先選擇一個基準值（Pivot），
再將資料分為小於 Pivot 與大於 Pivot 的兩部分，
接著對左右子陣列重複進行相同操作。

快速排序在平均情況下具有極佳的排序效能，
但效能表現會受到 Pivot 選擇影響。

**優點：**
- 平均效率高、實務上常被使用

**缺點：**
- 在最差情況下可能退化為 O(n²)

---

## 三、複雜度分析

### （一）時間複雜度

| 排序方法 | 最佳情況 | 平均情況 | 最差情況 |
|--------|--------|--------|--------|
| 氣泡排序 | O(n) | O(n²) | O(n²) |
| 選擇排序 | O(n²) | O(n²) | O(n²) |
| 插入排序 | O(n) | O(n²) | O(n²) |
| 合併排序 | O(n log n) | O(n log n) | O(n log n) |
| 快速排序 | O(n log n) | O(n log n) | O(n²) |

---

### （二）空間複雜度

| 排序方法 | 空間複雜度 | 是否需額外記憶體 |
|--------|----------|--------------|
| 氣泡排序 | O(1) | 否 |
| 選擇排序 | O(1) | 否 |
| 插入排序 | O(1) | 否 |
| 合併排序 | O(n) | 是 |
| 快速排序 | O(log n) | 是（遞迴堆疊） |

## 四、實驗設計與方法

### （一）實驗目的

透過實際程式執行五種排序演算法，
比較不同資料規模（n）下的排序時間，
觀察時間複雜度隨資料量增加的變化趨勢。

---

### （二）實驗設定

- 排序資料內容：隨機整數陣列
- 每一種排序法皆使用相同資料進行測試
- 測試資料筆數：
  - n = 100
  - n = 1,000
  - n = 5,000
  - n = 10,000
- 每組實驗重複多次並取平均值
